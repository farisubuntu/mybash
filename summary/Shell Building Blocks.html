Shell Building Blocks

Shell Functions are executed in the current shell context; no new process is created to interpret them.
See Chapter 11

The fork-and-exec scheme is only applied after the shell has analyzed input in the following way:

  • The shell reads its input from a file, from a string or from the user's terminal.

  • Input is broken up into words and operators, processing the quoting rules, see Chapter 3. These tokens are separated by metacharacters. Alias expansion is performed.

  • The shell parses (analyzes and substitutes) the tokens into simple and compound commands.

  • Bash performs various shell expansions, breaking the expanded tokens into lists of filenames and commands and arguments.

  • Brace expansion { }

expanding values between braces, such as file{1..3} into file1 file2 file3 Or a{d,c,b}e expands into 'ade ace abe'

  • Tilde expansion ~

expanding the ~ tilde symbol for the home directory into the path to the home directory, such as ~/effective-shell into /home/dwmkerr/effective-shell

  • Parameter and variable expansion $

expanding terms that start with a $ symbol into parameter values, such as $HOME into the value of the variable named HOME

  • Command substitution $(cmd)

evaluation of the contents of $(command) sequences, which are used to run commands and return the results to the shell command line

  • Arithmetic expansion $((exp)) , $[ expr ]

    > note spaces inside [ ]

evaluation of the contents of $((expression)) sequences, which are used to perform basic mathematical operations.

The expression is treated as if it were within double quotes, but a double quote inside the parentheses is not treated specially.

  • Word splitting (IFS)

    > If no expansion occurs, no splitting is performed.

once all of the previous operations are run, the shell splits the command up into 'words', which are the units of text that you can run loops over

The "IFS" (Internal Field Separator) variable plays an important role in controlling how fields in a string are separated. IFS defaults to a space, tab, and newline character, which means that, by default, fields in a string are separated by any combination of these characters.

  • Filename expansion

    Occur only after word splitting, unless the -f option has been set

The shell expands wildcards and special characters in pathnames, such as file*.txt into the set of files that are matched by the sequence.

> Bash scans each word for the characters "*", "?", and "[". If one of these characters appears, then the word is regarded as a PATTERN, and replaced with an alphabetically sorted list of file names matching the pattern.

  • Redirection is performed if necessary, redirection operators and their operands are removed from the argument list.

  • Commands are executed.

  • Check whether the command contains slashes. If not, first check with the function list to see if it contains a command by the name we are looking for.

  • If command is not a function, check for it in the built-in list.

  • If command is neither a function nor a built-in, look for it analyzing the directories listed in PATH. Bash uses a hash table (data storage area in memory) to remember the full path names of executables so extensive PATH searches can be avoided.

  • If the search is unsuccessful, bash prints an error message and returns an exit status of 127.

  • If the search was successful or if the command contains slashes, the shell executes the command in a separate execution environment.

  • If execution fails because the file is not executable and not a directory, it is assumed to be a shell script.

  • If the command was not begun asynchronously, the shell waits for the command to complete and collects its exit status.

  • Optionally the shell waits for the command to complete and collects its exit status.
